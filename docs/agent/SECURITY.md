# Security

- No guardar secretos, tokens, llaves privadas ni datos reales.
- No exponer PII en ejemplos.
- Usar emails y IDs ficticios.
- No documentar rutas internas sensibles como si fueran publicas.
- Todo webhook debe contemplar firma o mecanismo de autenticidad.
- Todo evento debe incluir trazabilidad sin revelar datos innecesarios.

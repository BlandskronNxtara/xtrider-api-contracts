# Release

## Pre-release

1. Revisar `CHANGELOG.md`.
2. Confirmar version SemVer.
3. Ejecutar validaciones.
4. Confirmar que no hay secretos ni datos reales.
5. Confirmar que breaking changes tienen plan de adopcion.

## Publicacion

La publicacion de SDKs o paquetes no esta implementada. Requiere aprobacion humana y ADR.
